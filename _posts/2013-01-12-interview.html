Javascript：

1. 闭包的理解，javascript的作用域（高频率）

先来回答javascript作用域的问题,javascript语言没有块级作用域，只有一条从内向外的作用域链，看这个例子

{% highlight javascript %} var scope = "global"; function CheckScope(){ console.log(scope); var scope = "local"; console.log(scope); } CheckScope(); {% endhighlight %}

输出的值是1：undefied 2:local

其实这个函数等价于下面这个函数（变量提升）

{% highlight javascript %} var scope = "global"; function CheckScope(){ var scope; console.log(scope); scope = "local"; console.log(scope); } CheckScope(); {% endhighlight %}

再来看个例子

{% highlight javascript %} var scope = "global"; function CheckScope(){ console.log(scope); console.log(scope2); var scope2 = "local2"; console.log(scope2); } CheckScope(); {% endhighlight %}

同样这个函数等价于下面这个函数（变量提升）

{% highlight javascript %} var scope = "global"; function CheckScope(){ var scope2; console.log(scope); console.log(scope2); scope2 = "local"; console.log(scope2); } CheckScope(); {% endhighlight %}

输出的值是1：global 2：undefied 3:local

从例子中我们可以看出：一种情况，函数里面的变量和函数外部变量同名时候,作用域顶端（函数代码块里面）先遍历查找变量的定义和赋值，如果找不到再向作用域末端（函数代码块外面）去查找；这样就形成了javascript特有的链式作用域。
