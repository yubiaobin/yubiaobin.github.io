初次运行 Git 前的配置

用户信息

第一个要配置的是你个人的用户名称和电子邮件地址。这两条配置很重要，每次 Git 提交时都会引用这两条信息，说明是谁提交了更新，所以会随更新内容一起被永久纳入历史记录：

{% highlight javascript %} $ git config --global user.name //设置全局用户名 {% endhighlight %} {% highlight javascript %} Example: $ git config --global user.name "yubiaobin" {% endhighlight %} {% highlight javascript %} $ git config --global user.email //设置全局用户邮箱 {% endhighlight %} {% highlight javascript %} Example: $ git config --global user.email yubiaobin@163.com {% endhighlight %}

Git create（git 创建命令）

Git 可以使用四种主要的协议来传输数据：本地传输，SSH 协议，Git 协议和 HTTP 协议。比较常用的就是SSH协议和HTTP协议。

{% highlight java %} $ git clone ssh:user@domail.com/repo.git //（克隆远程已存在的版本库） {% endhighlight %} {% highlight java %} $ git clone https:user@github.com/someone/repo.git //（克隆远程已存在的版本库） {% endhighlight %} {% highlight java %} $ git init //（创建本地的本版库并初始化——即增加.git隐藏文件来记录版本） {% endhighlight %}

Local changes（本地的更改命令）

{% highlight java %} $ git status //（本地目录状态） {% endhighlight %} {% highlight java %} $ git diff //（本地目录追踪文件更改的比较） {% endhighlight %} {% highlight java %} $ git add --all //（添加当前所有有过改动的文件） {% endhighlight %} {% highlight java %} $ git commit //（提交更改的记录到暂存区） {% endhighlight %} {% highlight java %} $ git commit -a //提交之前所有本地追踪文件的更改） {% endhighlight %} {% highlight java %} $ git commit --amend //查看上次提交的更改的部分） {% endhighlight %} {% highlight java %} $ git commit -m //（添加这次提交更改的文字说明） {% endhighlight %}

Commit history（提交历史）

{% highlight java %} $ git log //（从最近的提交开始显示所有的提交） {% endhighlight %} {% highlight java %} $ git reflog //（从最近的提交开始显示所有的提交） {% endhighlight %} {% highlight java %} $ git log -p //（特定文件的按时间显示提交历史） {% endhighlight %} {% highlight java %} $ git log --pretty=oneline //（单行显示所有的提交） {% endhighlight %} {% highlight java %} $ git blame //（什么时候谁更改了文件） {% endhighlight %}

Branches & Tags（分支和标识）

{% highlight java %} $ git branch //（列出所有现有的分支） {% endhighlight %} {% highlight java %} $ git checkout //（切换到这个分支） {% endhighlight %} {% highlight java %} $ git branch //（在当前版本库创建一个新的分支） {% endhighlight %} {% highlight java %} $ git branch --tracked //（在远程版本库创建一个新的跟踪的分支） {% endhighlight %} {% highlight java %} $ git branch -d //（删除一个本地的分支） {% endhighlight %} {% highlight java %} $ git tag //（标记最新的提交） {% endhighlight %} {% highlight java %} $ git push --tags //（发布本地所有的标记去远程本版库） {% endhighlight %}

Update & Publish（更新和发布）

{% highlight javascript %} $ git remote -v //（列出所有当前的远程配置） {% endhighlight %} {% highlight javascript %} $ git remote show //（显示远程版本库信息） {% endhighlight %} {% highlight javascript %} $ git remote add //（增加一个新的名为的远程版本） {% endhighlight %} {% highlight javascript %} $ git fetch //（从远程下载所有更改，但是不和本地合并） {% endhighlight %} {% highlight javascript %} $ git pull //（从远程下载所有更改，并且和本地合并） {% endhighlight %} {% highlight javascript %} $ git push //（发布本地的提交的更改去远程版本库） {% endhighlight %} {% highlight javascript %} $ git push : //（删除远程的版本库的一个分支） {% endhighlight %} {% highlight javascript %} $ git push --tags //（发布本地所有的标记去远程本版库） {% endhighlight %}

Merge & Rebase（合并，重定义）

{% highlight java %} $ git merge //（合并分支去当前版本） {% endhighlight %} {% highlight java %} $ git rebase //（重定义当前的版本） {% endhighlight %} {% highlight java %} $ git rebase --abort //（中止重定义） {% endhighlight %} {% highlight java %} $ git rebase --continue //（解决冲突之后继续） {% endhighlight %} {% highlight java %} $ git diff //（本地目录追踪文件更改的比较） {% endhighlight %}
