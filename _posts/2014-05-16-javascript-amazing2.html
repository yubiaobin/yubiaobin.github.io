Javascript的变量提升

先来看一段英文原文：In JavaScript, a variable can be declared after it has been used. In other words, a variable can be used before it has been declared. To understand this, you have to understand the term "hoisting". Hoisting is JavaScript's default behavior of moving all declarations to the top of the current scope (to the top of the current script or the current function).

翻译过来大致意思是：在Javascript语言中，变量的声明可以放在它的使用之后。换句话说，变量可以先使用后声明。为什么会这样呢？哦，原来一切都是提升（hoisting）机制在“作怪”。提升（hoisting）是Javascript语言默认就具有的一种将当前作用域内的所有声明都提升（hoisting）到最顶部的行为机制。

为了理解上面的含义，我们来看具体案例，此处定义一个函数 fn1：

{% highlight javascript %} function fn1(){ var a = "变量一"; var b = "变量二"; var c = "变量三"; } {% endhighlight %}

其实它会“摇身一变”成这样：

{% highlight javascript %} function fn1(){ var a,b,c; a = "变量一"; b = "变量二"; c = "变量三"; } {% endhighlight %}

变量提升（但是）

但是（注意这个但是）：变量提升 只是提升变量的声明，并不会把赋值也提升上来。

我们来跳坑：

{% highlight javascript %} var name = "jack"; function fn1(){ alert(name); } fn1(); // jack {% endhighlight %}

此处弹出jack不错，再来：

{% highlight javascript %} var name = "jack"; function fn1(){ alert(name); var name = "peter"; } fn1(); // undefined {% endhighlight %}

此处并不会弹出“peter", 而是弹出undified 有些人回叙会不解？ 为什么会这样呢？

我们把上面的函数改写一下（内容没有变化）：

{% highlight javascript %} var name = "jack"; function fn1(){ var name; alert(name); name = "peter"; } fn1(); // undefined {% endhighlight %}

释义：代码执行函数fn1，运行到执行alert语句的时候当前作用域（函数体内）只有变量name的定义，并没有赋值，所以alert语句打印出来的当然是undefined了。

然后，我们可以再次改写函数（内容有变化）：

{% highlight javascript %} var name = "jack"; function fn1(){ var name; alert(name); name = "peter"; alert(name); } fn1(); // undefined peter {% endhighlight %}

这个时候，执行函数fn1，运行到执行第一条alert语句的时候当前作用域（函数体内）只有变量name的定义，并没有赋值，所以弹出undefined了，接着给变量name赋值，所以执行第二条alert语句的时候。变量name的值为peter，所以接着会打印peter出来。
